## What This Project Is

A Kotlin Multiplatform (KMP) library that provides type-safe analytics event definitions for MEGA's mobile apps (Android, iOS, macOS, Windows). Events are defined via annotations and processed at compile time by a KSP annotation processor to generate event classes and stable JSON ID mappings.

## Build Commands

```bash
./gradlew build                        # Full build (includes KSP code generation)
./gradlew test                         # Run all tests
./gradlew :analytics-processor:test    # Run processor tests only (JUnit 5)
./gradlew :analytics-core:test         # Run core module tests
./gradlew clean build                  # Clean rebuild
```

## Architecture

Four modules:

- **analytics-annotations** — Annotation definitions (`@ButtonPressEvent`, `@ScreenViewEvent`, `@GeneralEvent`, etc.) used to declare events
- **analytics-core** — Base event types (`AnalyticsEvent`, identifiers), event generator, JSON mapping infrastructure
- **analytics-processor** — KSP annotation processor that reads annotated interfaces/classes and generates event code using KotlinPoet. Uses the Visitor pattern (one visitor per annotation type) and a factory-based ID generator
- **shared** — Consumer-facing module containing event definitions (annotated interfaces/classes in `event/`), the `Tracker` API, and generated JSON resource files with event ID mappings

## How to Add a New Event

1. Add an annotated interface or class in `shared/src/commonMain/kotlin/mega/privacy/mobile/analytics/event/` in the appropriate file (e.g., `ButtonPressEvents.kt`)
2. Simple events: use an `interface` — generates a singleton `object`
3. Events with runtime parameters: use a `class` with constructor params — generates a `data class`. Use `@StaticValue("value")` for compile-time constants in the payload
4. Run `./gradlew build` to trigger KSP generation
5. **Commit both the new annotation and the updated JSON file in `shared/src/commonMain/resources/`** — JSON files contain stable event IDs; if not committed, IDs can change and corrupt analytics data

## Critical: JSON Resource Files

The JSON files in `shared/src/commonMain/resources/` (e.g., `ButtonPressEvent.json`) map event names to unique numeric IDs. These are generated by KSP and **must be committed to version control**. Changing or losing these files corrupts analytics data because event IDs must remain stable.

## Event ID Formula

`platformIdentifier + eventTypeIdentifier + uniqueId + appIdentifier`

Event type offsets: ScreenView=0, TabSelected=1000, ButtonPress=2000, DialogDisplayed=3000, Navigation=4000, MenuItem=5000, Notification=6000, General=7000, ItemSelected=8000.

## Annotation Types

| Annotation | Required Params | Optional Params |
|---|---|---|
| `@ButtonPressEvent` | `buttonName` | `screen`, `dialog` |
| `@ScreenViewEvent` | — | — |
| `@TabSelectedEvent` | `screenName`, `tabName` | — |
| `@DialogDisplayedEvent` | `dialog` | `screen` |
| `@NavigationEvent` | `destination`, `navigationElementType` | — |
| `@MenuItemEvent` | `menuItem`, `menuType` | `screen` |
| `@NotificationEvent` | — | — |
| `@GeneralEvent` | — | — |
| `@ItemSelectedEvent` | — | — |
| `@GestureEvent` | — | — |
| `@LegacyEvent` | `eventId` | — |

Use `@Exclude` to skip generation for a class/file.

## Testing

Processor tests use kotlin-compile-testing to verify generated code output. Tests are in `analytics-processor/src/jvmTest/`. Assertions use Google Truth. Mocking via Mockito Kotlin.

## Publishing

Artifacts publish to Artifactory via `./gradlew artifactoryPublish`. Requires `ARTIFACTORY_USER` and `ARTIFACTORY_ACCESS_TOKEN` environment variables. Versioning is timestamp-based.
